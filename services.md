## Run Bundle — The Differentiator

Every automation I build produces a **Run Bundle**: a complete audit package containing:
- **Inputs:** All source data and parameters
- **Outputs:** All generated files and reports
- **Logs:** Complete execution trace with timestamps
- **Manifest:** Checksums and inventories for integrity verification

Every Run Bundle is:
- **Self-contained:** No external dependencies to understand what happened
- **Reproducible:** Same inputs = same outputs
- **Auditable:** Complete traceability for compliance and debugging
